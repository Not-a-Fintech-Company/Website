## P0 — Critical (fix this week)

### P0-1. GSC verification gap

`notafintech.co` is not in any Google Search Console property reachable from the system-level OAuth. The credentials at `~/.config/claude-seo/google-api.json` are scoped to a different Google project (`ayso13-seo`).

**Action:** Either add `sc-domain:notafintech.co` under the same Google account, or supply the OAuth credentials of whichever account verified notafintech.co in GSC. Without this you have zero search performance visibility post-migration.

**Also:** Update the GA4 numeric property ID — currently set to `307558725` (ayso13.org), not the notafintech property behind `G-VJXL7WCFNM`. Find the numeric ID at analytics.google.com → Admin → Property settings.

### P0-2. Post-migration indexation status is unknown

The May 13–16 redesign changed nothing about URLs (`_redirects` preserves Jekyll-era 301s), but you can't confirm Google has re-crawled.

**Action:** Once GSC verification (P0-1) is in place, submit `https://www.notafintech.co/sitemap-index.xml` manually in GSC, then run URL Inspection on `/`, `/tools/term-loan/`, and one guide page.

### P0-3. Empty stub sections in a published YMYL guide

`/guides/how-to-launch-a-card-product/` has `<h4>Underwriting</h4>`, `<h4>Collections</h4>`, and `<h4>Marketing ToDos</h4>` with no body content. On a financial-services guide, published-but-empty headings are a direct trust-rater failure signal.

**Action:** Either fill the sections or remove the empty headings + TOC anchors.

### P0-4. Zero `publishedAt` / `updatedAt` on any page

The schema supports both fields (verified in `src/content.config.ts`) but they're empty everywhere. AI search and Google freshness signals both penalize undated content, and Article schema (P1-5 below) requires `datePublished` to be valid.

**Action:** Populate the dates across all content files; render the `updatedAt` in `ArticleLayout`.

### P1-11. Stub Tools pages indexable with thin content

`/tools/bank-account/` and `/tools/credit-card/` are 40-word "Coming soon" pages, in the sitemap, no `noindex`. Two options:

- **A:** Set `draft: true` on the frontmatter (suppresses from sitemap + indexing).
- **B (recommended):** Expand to 150–250 words explaining what the tool will model, with a link to the corresponding Google Sheets as interim utility.

### P1-12. No per-article bylines

`/about/` lists 8 named contributors with LinkedIn, but no individual guide is attributed. Google QRG asks "who is responsible for this content?" at the page level.

**Action:** Add `author` field to frontmatter schema; render in `ArticleLayout`. Even one attributed author per guide closes the gap.

## P2 — Medium (this month)

### P2-13. Stronger E-E-A-T disclaimer on guides

YMYL-adjacent financial content currently has none. The BSA policy template has one ("this is a sample, customize it") — replicate the pattern as a site-wide footer note or per-guide block.

### P2-14. Headings as questions on guide pages

Convert topic labels ("Networks", "Compliance", "Risk") to question form ("How do you choose a card network?"). Direct match for AI search query patterns; lifts AI Overviews / Perplexity citation rate.

### P2-15. Resources page is a raw link dump

Bare URLs (some MDX-incompatible), no editorial context, multiple stale references (2017 GetFinal posts, defunct Synapse). Rewrite as `[Title](url)` with one-line annotations; cull or annotate >3-year-old links.

### P2-16. Switch to Latin-only font subsets

`@fontsource-variable/eb-garamond/index.css` ships Cyrillic + Greek + Vietnamese subsets unused. Switch to `latin.css` to drop CSS bundle by ~50 KB.

### P2-17. Add `<link rel="preload">` for the two critical font files

Self-hosted variable fonts are excellent, but with no preload they're discovered only after CSS parses. Cuts LCP by 200–400ms on first-visit mobile.

### P2-18. Add fallback `@font-face` descriptors

(`size-adjust`, `ascent-override`, `descent-override`) for EB Garamond fallback. Suppresses CLS from `font-display: swap` on multi-line h1s.

### P2-19. IndexNow setup

Not configured. For Bing + Yandex + Naver instant URL submission on publish. Generate UUID key, place at `public/<uuid>.txt`, POST changed URLs from a Cloudflare Pages deploy hook.

### P2-20. Extend `_astro/*` cache TTL to immutable

Currently `max-age=14400` (4 hours). Astro content-hashes these filenames; safe to set `max-age=31536000, immutable` via `public/_headers`.
