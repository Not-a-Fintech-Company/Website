# .impeccable.md

Design context for `notafintech.co`. This file feeds the impeccable, audit, critique, polish, and related design skills.

## Design Context

### Users
Fintech founders, operators, and builders. Audience includes early-stage solo founders weighing whether to launch a card product or stand up a challenger bank, plus operating-team folks (compliance, ops, engineering) who need policy templates and financial model references. Context of use: weekday daytime, on a laptop, between meetings — people skim guides for one specific answer, copy a sheet/template, and leave. Less commonly: late-night research sessions when someone is trying to understand a domain they're new to.

### Use Cases
- Read a long-form guide on a specific fintech topic (`/guides/*`)
- Copy a Google Sheets financial model template (`/models/*`)
- Reference a compliance policy template, e.g. BSA/AML (`/docs/*`)
- (Future) Run interactive Preact calculators (`/tools/*`)
- Quickly hop between related topics via search or sidebar

The site is a *reference resource*, not a marketing site, not a SaaS app. People rarely "convert" — they read, copy, and bookmark.

### Brand Personality
**Three words**: open-source · authoritative · editorial.

- **Open-source**: All content under CC0 1.0; the site itself is open on GitHub; community-contributed. The tone should never feel proprietary, gated, or salesy.
- **Authoritative**: This is grown-up financial-services material — fintech founders deserve careful, sourced writing, not breezy startup-blog energy.
- **Editorial**: Think *Stripe Press*, *The Generalist*, *Matt Levine's column*. Confident typographic hierarchy, restrained color, generous whitespace. Says "this is well-researched writing for serious people."

The interface should evoke *quiet competence and trust* — not delight, not playfulness. A user should feel they're reading something dependable.

### Aesthetic Direction
**Locked in the migration spec (2026-05-12) and shipped in the redesign branch.**

- **Theme**: Both light and dark, with explicit user toggle persisted to localStorage. Light is the default (most users on weekday daytime, laptop). Dark mode honors `prefers-color-scheme` on first visit if no preference set.
- **Palette** (warm, editorial — not the typical fintech blue; values reflect the May 13 contrast hardening):
  - `paper` `#FAF7F2` (light) / `#15110E` (dark) — page background, warm off-white / warm near-black
  - `ink` `#1A1714` (light) / `#F4EFE7` (dark) — primary text
  - `accent` `#7A2E2A` (light) / `#CD7572` (dark) — deep **oxblood**, used sparingly (links, focus rings, brand). Dark variant raised to ~5.6:1 on dark paper (was failing AA)
  - `accent-hover` `#5C2220` (light) / `#DC8B89` (dark)
  - `rule` `#D2C5B0` (light) / `#3C3128` (dark) — dividers, subtle underlines; darkened from prior values for stronger UI weight
  - `muted` `#5A534D` (light, ~6.5:1) / `#9E968F` (dark) — secondary text
  - `surface` `#FFFFFF` (light) / `#1F1813` (dark); `surface-2` `#EFE8D9` / `#2B221A` — card and section-tinted surfaces
- **Typography** (revised 2026-05-15 to break out of Fraunces+Inter AI-monoculture):
  - Display / headings: **EB Garamond** (variable, OFL, self-hosted) — classical Garalde with the gravitas of an SEC document or central bank report. True italic forms (cursive, distinct from roman). Used by *The Generalist* for body type.
  - Body & UI: **Hanken Grotesk** (variable, OFL, self-hosted) — warm humanist sans, not Inter, less common in modern AI editorial. Clear UI legibility with subtle character.
  - Numeric / monospace: **JetBrains Mono** (kept) — utilitarian, not on reflex-reject list.
- **Iconography**: Lucide (clean line icons, restrained use)
- **Visual rhythm**: 4pt spacing scale, prose measure `max-width: 68ch`, body line-height ~1.7

### References
- **Stripe Press** (https://press.stripe.com) — confident editorial layout, serif headlines, restrained color
- **The Generalist** (https://www.generalist.com) — editorial finance writing, typographic hierarchy
- **Matt Levine's column at Bloomberg** — long-form authority feel

### Anti-references
- Generic fintech-startup look: royal blue + glassmorphism + gradient buttons + cyan accents on dark
- "Modern AI" tells: Inter everywhere with no display pairing, purple-to-blue gradients, card-grid landing pages, big metric heroes, side-stripe alerts
- Salesy SaaS aesthetic: pricing tiles, "Trusted by" logo grids, hero CTAs over screenshots

### Design Principles
1. **Editorial first, app second**. Reading is the primary job. Typography, measure, and rhythm matter more than density. UI chrome stays out of the way.
2. **Quiet, not loud**. Restrained color (one accent), restrained motion, restrained ornament. A single oxblood link in a sea of warm ink is the point — flood it with color and the affordance disappears.
3. **Source-of-truth over polish-of-the-day**. CC0 content needs to be readable, copyable, and citable. Decorative effects that obstruct copy/paste, screenshots, or printability are wrong by default.
4. **Two patterns, used consistently**. Inline-prose links are oxblood and underlined (loud, because sparse). List/nav links are ink with a rule-colored underline that intensifies to oxblood on hover (quiet, because dense). One rule per context.
5. **Dark mode is a setting, not a redesign**. Same hierarchy, same density, same affordances — palette only.

### Accessibility
- Target: **WCAG AA**, fully met.
- Visible focus rings (`outline: 2px solid var(--color-accent); outline-offset: 2px`), provided globally.
- Theme toggle works without page reload; first-paint script reads `localStorage` before any CSS to prevent flash of wrong theme.
- Mobile touch targets ≥ 44px on interactive UI (header buttons, mobile menu items).
- Skip-link / `<main id="main">` landmark present.

### Tech Stack Constraints
- Astro 6 + Tailwind v4 (CSS-first config via `@theme`)
- Preact for islands (search modal; future calculators)
- Pagefind for search (custom modal UI, post-build index)
- Self-hosted fonts via `@fontsource[-variable]/*`
- Cloudflare Pages deploy from `main`
